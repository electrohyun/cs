# HTTP vs HTTPS

## 잘 알고 있는지

- 조금

## 내가 아는 만큼 설명

- HTTP는 하이퍼텍스트 트랜스퍼 프로토콜이다. 요청과 응답 형식으로 이루어진다. HTTPS는 보안이 있는 거다(?).
- HTTPS를 써야 쿠키 전달이 되고 그런 게 있었던 기억이 난다.
- 아는게 별로 없는 것 같다.

## 답을 본 뒤 알게 된 것

**핵심 한 줄**

- HTTP는 평문(암호화 X) 통신, HTTPS는 여기에 **TLS(SSL) 암호화 계층**을 씌운 것. 즉 `HTTPS = HTTP + TLS`.

**HTTP란**

- HyperText Transfer Protocol. 웹에서 클라이언트(브라우저)와 서버가 요청(Request)/응답(Response) 형식으로 데이터를 주고받는 애플리케이션 계층 프로토콜.
- 상태를 저장하지 않는 **stateless** 프로토콜이다. (그래서 상태 유지를 위해 쿠키·세션을 쓴다.)
- 데이터를 **평문(plain text)** 으로 전송한다 → 중간에서 가로채면(스니핑) 내용이 그대로 노출되고, 위·변조(중간자 공격, MITM)도 가능하다.

**HTTPS란**

- HTTP over TLS. HTTP 메시지를 **TLS로 암호화**해서 주고받는다. (예전엔 SSL이라 불렀고 지금은 TLS가 표준. 관습적으로 SSL/TLS라 부름)
- TLS가 보장하는 3가지:
  - **기밀성(암호화)**: 가로채도 내용을 못 읽는다.
  - **무결성**: 중간에 데이터가 변조되면 감지된다.
  - **인증**: 서버가 진짜 그 서버가 맞는지 **인증서(CA가 서명)** 로 검증한다.

**암호화 방식 — 대칭키 하나가 아니다 (핵심 포인트)**

- **비대칭키(공개키/개인키)** 로 안전하게 "대칭키를 교환"하고, 실제 데이터 통신은 빠른 **대칭키**로 암호화한다. (둘을 섞어 쓰는 하이브리드 방식)
- 이유: 비대칭키는 안전하지만 느리고, 대칭키는 빠르지만 키 전달이 위험 → 장점만 조합.

**TLS 핸드셰이크 흐름 (간단 버전)**

1. 클라이언트가 접속 → 서버가 **인증서(공개키 포함)** 를 보냄.
2. 브라우저가 인증서를 **CA(신뢰된 인증기관)** 서명으로 검증 (도메인·유효기간·발급자 확인).
3. 공개키로 안전하게 **세션키(대칭키)** 를 합의(교환).
4. 이후 실제 HTTP 데이터는 세션키로 **대칭키 암호화**하여 주고받음.

**부가 사실**

- 기본 포트: HTTP = **80**, HTTPS = **443**.
- `Secure` 속성이 붙은 쿠키는 HTTPS(암호화 연결)에서만 전송된다 → 내가 기억한 "HTTPS 쿠키" 얘기는 이걸 어렴풋이 떠올린 것.
- SEO·브라우저 정책상 HTTPS가 사실상 필수 (HTTP는 "안전하지 않음" 경고 표시).

## 틀렸거나 부족했던 부분

## 면접용 1분 답변

- HTTP는 하이퍼텍스트 트랜스퍼 프로토콜의 약자인데요, HTTPS는 여기에 SSL이라고 불리기도 했던 TLS 암호화 계층이 더해진 방식입니다.
- HTTP는 기본적으로 클라이언트의 요청 그리고 서버의 응답으로 작동하며, 상태가 없는 stateless입니다. 이런 점은 쿠키나 세션을 통해 보완합니다.
- 데이터를 평문으로 보내기 때문에 스니핑에 취약하다는 단점도 있고요.

- TLS는 기밀성, 무결성, 인증 3가지를 보장합니다. 기밀성은 가로채도 내용을 못 읽게, 무결성은 변조되면 알 수 있게, 인증은 진짜 그 서버가 맞는지 인증서로 검증합니다.

- 암호화 방식이 하이브리드 방식이에요. 대칭키는 빠르지만, 비대칭키는 뚫기 어렵죠. 그래서 대칭키를 비대칭키로 교환하고, 실제 통신은 대칭키로 합니다.

- TLS 핸드쉐이크는 (1) 클라이언트가 접속하면 서버가 공개키 포함된 인증서를 보냅니다.
  (2) 브라우저가 인증서를 CA의 서명으로 검증합니다. 도메인, 유효기간, 발급자를 확인합니다.
  (3) 공개키로 안전하게 세션키(대칭키)를 교환하고,
  (4) 이제 실제 HTTP 데이터는 세션키로 대칭키 암호화하여 주고받습니다.

- HTTPS를 쓰는 이유는, Secure 속성이 붙은 쿠키의 경우 반드시 HTTPS 환경에서만 전송되는 이유도 있기 때문입니다.
