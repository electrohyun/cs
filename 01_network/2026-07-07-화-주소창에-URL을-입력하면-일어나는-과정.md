# 주소창에 URL을 입력하면 일어나는 과정

## 잘 알고 있는지

- 🥺

## 내가 아는 만큼 설명

주소창에 주소를 입력하면 브라우저가 해당 주소를 찾고,
서버에 요청을 보내서 HTML, CSS, JS 같은 파일을 받아온다.
그 후 브라우저가 화면에 보여준다.

## 답을 본 뒤 알게 된 것

- URL을 먼저 해석한다.
- 도메인은 바로 서버 주소가 아니라 DNS를 통해 IP 주소로 변환된다.
- 서버와 통신하기 위해 TCP 연결을 맺는다.
- HTTPS라면 TLS handshake를 통해 암호화 연결을 설정한다.
- 브라우저는 HTTP 요청을 보내고 서버는 HTML, CSS, JS 등을 응답한다.
- 브라우저는 HTML을 파싱해서 DOM을 만들고, CSSOM과 합쳐 렌더 트리를 만든다.
- 이후 Layout, Paint, Composite 과정을 거쳐 화면에 표시한다.

## 틀렸거나 부족했던 부분

- DNS 과정을 몰랐다.
- TCP와 TLS의 순서를 몰랐다.
- HTTP 요청 이후 브라우저 렌더링 과정이 있다는 것을 자세히 몰랐다.
- 단순히 “서버에서 페이지 받아옴” 정도로만 알고 있었다.

## 면접용 1분 답변

사용자가 주소창에 URL을 입력하면 브라우저는 먼저 URL을 해석합니다.  
그 다음 DNS를 통해 도메인에 해당하는 IP 주소를 찾고, 해당 서버와 TCP 연결을 맺습니다.  
HTTPS라면 TLS handshake를 통해 암호화 연결을 설정한 뒤 HTTP 요청을 보냅니다.  
서버는 HTML, CSS, JavaScript 등의 리소스를 응답하고, 브라우저는 이를 파싱해 DOM과 CSSOM을 만든 뒤 렌더 트리를 구성합니다.  
이후 Layout, Paint, Composite 과정을 거쳐 최종적으로 화면에 웹페이지를 보여줍니다.

## 한 줄 요약

URL 입력 → DNS → TCP/TLS 연결 → HTTP 요청/응답 → 브라우저 렌더링
