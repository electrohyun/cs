- 프로토타입(Prototype)
- JavaScript에서 모든 객체는 다른 객체를 참조하는 링크를 하나 갖는다. 그 링크가 가리키는 객체를 프로토타입이라고 부른다.
- "어떤 객체가 프로퍼티를 조회할 때, 자기 자신에게 없으면 대신 찾아보도록 위임하는 다른 객체."

- 역할: 어떤 프로퍼티를 조회했는데, 없으면 -> 이 링크를 타고 프로토타입에게 가서 찾는다. 거기도 없으면 그 프로토타입의 프로토타입으로... ??

  - 취약점도 있다: 프로토타입 오염, this의 미끄러움, 런타임에 prototype 덮어쓰기.

  - 문법적 설탕 관계
  - 같은 실체, 다른 표현

```js
// ── 1. 평범한 객체 ──
const obj = { name: "두부" };

typeof obj; // "object"
obj(); // ❌ TypeError: obj is not a function  (못 부름)
obj.prototype; // undefined  (창고 없음)

// ── 2. 함수 ──
function fn() {
  return "안녕";
}

typeof fn; // "function"
fn(); // "안녕"  ✅ (부를 수 있음!)
fn.prototype; // {}  ✅ (창고 딸려있음!)
fn.name; // "fn"  (프로퍼티도 담김 — 객체니까)
```

```
obj (평범한 객체)
 └─ name: "두부"        ← 프로퍼티 담기만 가능

fn (함수 = 객체 + α)
 ├─ name: "fn"          ← 프로퍼티 담기 (obj랑 같음)
 ├─ color: "빨강"       ← 프로퍼티 담기 (obj랑 같음)
 ├─ ()로 호출 가능       ← α (obj엔 없음)
 └─ prototype: {}       ← α 창고 (obj엔 없음)
```

  - object는 function의 좀 떨어진 부모 느낌.
  - object는 만물의 조상.

Function은 생성자잖아.
그니까 Function.prototype 문다매. (생성자는 Function.prototype 무는 거임)

그리고, Array.prototype은 창고잖아.
그니까 Object.prototype을 문대. (창고는 Object.prototype 무는 거임)

=====

이제, Function.prototype은 함수처럼 생겼지만(typeof가 function), 정체는 창고잖아.
그니까 얘도 Object.prototype 문다. (창고니까. Array.prototype이랑 똑같음)
→ "함수냐"에 낚이면 폭발함. "창고냐 생성자냐"로 보면 안 폭발함.
→ Function.prototype은 이름에 .prototype 붙었으니 창고 → Object.prototype.

- 인스턴스의 [[Prototype]] → 자기를 만든 생성자의 창고
- 생성자의 [[Prototype]] → Function.prototype (전부 예외없이)
- 창고의 [[Prototype]] → Object.prototype (단 Object.prototype만 null)

### 표

| 대상               | 정체     | .prototype (창고 딸림?) | [[Prototype]] (부모) |
| ------------------ | -------- | ----------------------- | -------------------- |
| a                  | 인스턴스 | 없음                    | Array.prototype      |
| Array              | 생성자   | Array.prototype         | Function.prototype   |
| Function           | 생성자   | Function.prototype      | Function.prototype   |
| Object             | 생성자   | Object.prototype        | Function.prototype   |
| Array.prototype    | 창고     | 없음                    | Object.prototype     |
| Function.prototype | 창고     | 없음                    | Object.prototype     |
| Object.prototype   | 창고     | 없음                    | null                 |

- class / extends의 실체

```js
class Dog {
  constructor(name) {
```

- extends

```js
class Puppy extends Dog {
  play() {
    return "논다";
  }

const p = new Puppy("두부");
p.bark(); // "두부 멍"  ← Dog한테서 물려받음

```

- `Puppy.prototype ─[[Prototype]]─> Dog.prototype`

```js
// class extends 가 하는 일을 날것으로:
```

- 섀도잉

```js
function Dog(name) {
  this.name = name;
}
Dog.prototype.bark = function () {
  return "창고 멍";
};

const a = new Dog("두부");
a.bark = function () {
  return "몸통 멍";
}; // 몸통에도 bark!

a.bark(); // ?
```

- 창고에도 bark 있지만, 몸통부터 조회 시작함 -> 찾으면 멈춘다!!
- 이렇게 자식 것이 부모 것을 가리는 것이 섀도잉. (사라진 건 아님. 가린 것 뿐)

- 섀도잉 vs 오버라이드
- 겉보기 결과는 같음: 자식(가까운) 게 부모(먼) 걸 이김.
- 속은 정반대:
  - 오버라이드(Java/C++): 부모 걸 "교체". 컴파일 때 확정(vtable). 부모 건 super로만 접근. 정적.
  - 섀도잉(JS): 부모 것도 자식 것도 "둘 다 살아있음". 체인에서 가까운 걸 먼저 만나 멈출 뿐. 먼 건 가려진 것뿐 삭제 아님. 동적.
- 결정적 차이: 섀도잉은 가림막(자식 것) 지우면 부모 게 부활. 오버라이드는 불가능.
  - delete Puppy.prototype.bark → p.bark()가 다시 부모 "개 멍" 나옴.
- 철학: 오버라이드=교체(정적, 타입안전) / 섀도잉=우선순위·공존(동적, 유연하지만 버그 숨음).
- 한 줄: JS class 상속은 오버라이드처럼 보여도 속은 섀도잉. 설탕 아래는 프로토타입.

- constructor

```js
function Dog(name) {
  this.name = name;
}
const a = new Dog("두부");

a.constructor; // Dog 함수
a.constructor === Dog; // true
```

- constructor는 Dog.prototype에 살음.
- 함수를 만들면 엔진이 .prototype 창고를 자동으로 만들어줌 (2번). 그때 창고 안에 constructor도 자동으로 넣어줌.

```js
a.constructor === Dog; // true
[].constructor === Array; // true
({}).constructor === Object; // true
```

```js
function Dog() {}
Dog.prototype = { bark() {} }; // 창고를 통째로 새 객체로 교체

const a = new Dog();
a.constructor === Dog; // false! ← Object가 나옴
```

- 함정: 창고를 통째로 갈아끼우면 엔진이 자동으로 넣어준 constructor가 날아감.

- constructor = 인스턴스 → 자길 만든 생성자를 가리키는 링크
- 위치: 몸통 아니라 생성자.prototype 창고에 있음 (체인 타고 빌려봄)
- 자동: 함수 만들면 엔진이 창고에 넣어줌 (생성자↔창고 상호 참조)
- 함정: prototype 통째로 교체하면 끊김 → 수동 복구 필요 (class는 자동)


- 공격 시나리오
- **proto** 통로로 Object.prototype에 심으면 모든 객체가 isAdmin=true로 보임. 실제 CVE 단골.

- 원인: 트레이드오프
- 장점: 창고 1벌 공유 → 메모리 효율.
- 단점: 창고 1벌 공유 → 하나 오염되면 전부 감염.
- 같은 구조(공유)가 장점이자 취약점.

```js
const safe = Object.create(null); // 조상 없는 객체, 창고 링크 자체 없음
safe.__proto__; // 일반 프로퍼티일 뿐, 통로 안 열림
```

- 방어하는 법
- 사용자 입력에서 **proto**, constructor 키 필터링.
- Object.freeze(Object.prototype) 로 창고 얼려서 못 심게.

- 공유가 효율의 근원이자 취약점의 근원. 프로토타입의 빛과 그림자.

=====

- 부트스트랩

```
Object(함수)의 [[Prototype]]     = Function.prototype
Function.prototype의 [[Prototype]] = Object.prototype
Object.prototype이 누구 창고?      = Object(함수)의 .prototype
```

- 즉:

```
Object ──[[Prototype]]──> Function.prototype ──[[Prototype]]──> Object.prototype
  ↑                                                                  │
  └────────────────── 이게 Object의 .prototype ──────────────────────┘
```

- 서로가 필요로 하는데 어떻게 세팅함? 닭-달걀 같은 건데, JS 엔진이 시동 걸 때(부트스트랩) 특별하게 처리.

```
1단계: 빈 껍데기들을 먼저 다 만들어 (링크 없이)
   Object.prototype  = {} (일단 빈 거)
   Function.prototype = {} (일단 빈 거)
   Object = 함수 (일단 만듦)
   Function = 함수 (일단 만듦)

2단계: 그 다음 링크를 서로 꽂아
   Object의 [[Prototype]] = Function.prototype
   Function.prototype의 [[Prototype]] = Object.prototype
   ... 전부 연결
```

- 해결: 엔진이 부팅 때 "다 만들고 → 나중에 링크" 로 세팅 (일반코드 못 하는 특권)
- 조회 안전: [[Prototype]] 선만 타면 여전히 null로 끝 → 무한루프 없음

## 면접용 1분 답변

- Q. "자바스크립트의 프로토타입에 대해 설명해주세요."
- A. 자바스크립트의 모든 객체는 [[Prototype]]이라는 내부 슬롯을 가지며, 이 슬롯은 다른 객체를 참조하는 링크입니다. 어떤 프로퍼티를 조회했을 때 자기 자신에게 없으면 이 링크를 타고 부모 객체로 올라가며 찾고, 이 연결을 프로토타입 체인이라고 합니다. 체인은 Object.prototype을 거쳐 최종적으로 null에서 끝납니다. 자바스크립트는 이 방식으로 상속을 구현합니다. 예를 들어 배열의 map이나 push 같은 메서드는 각 인스턴스가 아니라 Array.prototype이라는 창고에 한 벌만 존재하고, 모든 배열이 링크로 공유해서 빌려 씁니다. 덕분에 인스턴스를 아무리 많이 만들어도 메서드는 메모리에 한 벌만 존재해 효율적입니다. class 문법도 결국 이 프로토타입 위에 씌운 문법적 설탕이며, 내부적으로는 생성자 함수와 프로토타입 링크로 동작합니다.

- Q. (꼬리질문) "그럼 .prototype과 [[Prototype]]은 어떻게 다른가요?"
- A. .prototype은 함수만 가지는 프로퍼티로, 그 함수로 new 했을 때 인스턴스가 참조하게 될 창고입니다. [[Prototype]]은 모든 객체가 가지는 내부 슬롯으로, 프로퍼티 조회 시 실제로 타고 올라가는 링크입니다. 즉 .prototype은 "남에게 빌려줄 창고", [[Prototype]]은 "내가 부모에게서 빌리는 링크"로, 방향이 반대입니다. new가 실행될 때 생성자의 .prototype을 읽어 인스턴스의 [[Prototype]]에 연결해주는 관계입니다.

## 한 줄 요약

- 프로토타입 = "없으면 찾아갈 부모 객체" 링크([[Prototype]] 슬롯). 조회 시 체인 타고 올라가며 찾고 Object.prototype 거쳐 null로 끝난다. 메서드는 창고(.prototype)에 1벌 두고 모든 인스턴스가 공유 → 메모리 효율. class는 그 위의 문법적 설탕.
